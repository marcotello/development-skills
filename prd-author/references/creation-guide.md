# High-Integrity PRD Creation Guide: The Architectural-PM Method

### Phase 1: Context & Strategic Alignment
* **Step 1: Mission and Persona Definition:** Define if the product is a "simplified companion" or a "high-performance professional tool."
* **Step 2: External Benchmarking & Gap Analysis:** Identify specific "Pain Points" in the current market to solve.

### Phase 2: Core Identity & Constraints
* **Step 3: Strategic Branding:** Brainstorm names that align with the technical "vibe."
* **Step 4: The Technical "Anchor":** Establish standards for data integrity (e.g., specific numeric precision, data isolation).

### Phase 3: The Architecture Foundation
* **Step 5: Stack Alignment:** Explicitly choose frameworks supporting long-term scalability.
* **Step 6: Data Integrity Rules:** Document standards for operations (e.g., "Atomic transactions for state changes").

### Phase 4: Functional Decomposition
* **Step 7: Requirement Hierarchy:** Move from the Executive Summary to specific feature modules.
* **Step 8: AI & Interface Integration:** Specify if the system returns raw data or structured JSON for dynamic rendering.

### Phase 5: The Roadmap
* **Step 9: Logical Sequencing:** Group features into Epics based on the project lifecycle.
* **Step 10: Atomic Breakdown:** Break Epics into User Stories and then into "Units of Work."
